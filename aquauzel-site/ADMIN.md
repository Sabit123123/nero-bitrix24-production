# AquaUzel — админка товаров

Закрытая страница управления товарами. Публичный каталог и прайс формируются
из тех же данных. Бэкенд — Supabase (БД + вход + хранилище фото). Настройка
описана в [`supabase/README.md`](./supabase/README.md).

## Что изменено
- Добавлена закрытая страница **`/admin.html`** с входом по паролю (Supabase Auth — серверная проверка, без пароля в коде).
- Единый источник данных — таблица `products` в Supabase.
- Публичный **каталог** и **прайс** теперь строятся из товаров (с откатом на старый статический прайс, пока Supabase не настроен — сайт не ломается).
- Кнопка прайса теперь выгружает **CSV из актуальных данных** (открывается в Excel; для PDF — печать браузера, Ctrl/Cmd+P → «Сохранить как PDF»).
- Текущие 572 позиции перенесены в новый формат (миграция).

## Созданные/изменённые файлы
- `admin.html`, `assets/js/admin.js`, `assets/css/admin.css` — админка.
- `assets/js/store.js` — слой данных (чтение + CRUD + загрузка фото).
- `assets/js/supabase-config.js` — конфиг (URL + anon-ключ; заполняете вы).
- `supabase/schema.sql` — таблица, RLS, хранилище.
- `supabase/seed-products.sql` — миграция текущих товаров (генерируется `build/migrate_to_products.py`).
- `supabase/README.md`, `.env.example` — настройка.
- Изменены: `index.html` (подключение Supabase, кнопка прайса), `assets/js/app.js` (загрузка товаров, CSV).

## Как открыть админку
- Локально: `https://aquauzel.kz/admin.html` (или `…/admin.html` на любом домене сайта).
- Прямая ссылка с сайта не ведёт — страница закрыта (`noindex`), вход только по паролю.

## Вход и пароль
- На `/admin.html` вводится **только пароль**. Служебный email подставляется
  автоматически из `supabase-config.js` → `adminEmail` (по умолчанию `admin@aquauzel.kz`).
- Пароль задаётся при создании администратора в Supabase → Authentication → Users; меняется там же.
- В коде/репозитории пароля нет — его проверяет Supabase (публичный репозиторий его не раскрывает).

## Пасхалка «Вова»
- Публичный сайт: 20 кликов по логотипу — «Вова» выглядывает снизу. Картинка всегда оригинальная (заменить нельзя).
- Админка: 20 кликов по заголовку «Товары» — «Вова» выглядывает; пока он виден, клик по нему открывает выбор файла и заменяет картинку (только в этом браузере, у покупателей не меняется).

## Добавить товар
1. Войдите в `/admin.html`.
2. Кнопка **«+ Добавить товар»** → появится пустая карточка.
3. Заполните поля → **«Сохранить»** (появится уведомление «Сохранено»).

## Добавить фото (с телефона/компьютера)
- В карточке нажмите **«Загрузить фото»** → выбор источника:
  - на телефоне — камера или галерея;
  - на компьютере — выбор файла.
- Фото проверяется (только изображения, до 8 МБ), сжимается в браузере и грузится в хранилище. Показывается превью. Не забудьте **«Сохранить»**.

## Удалить товар
- Кнопка **«Удалить»** в карточке → подтверждение → товар удаляется из БД, из публичного каталога/прайса; фото-файл по возможности удаляется.

## Редактировать товар
- Измените поля → **«Сохранить»**. Публичный каталог/прайс обновятся автоматически (без правки кода).
- Галочка **«В наличии»** — снятая помечает товар как «Под заказ» в публичном прайсе.

## Как формируется прайс
- Из таблицы `products`: группировка по категориям → таблица с колонками
  Наименование / Упаковка / Характеристики / Наличие / Цена.
- Поиск и категории на публичной странице сохранены.
- Кнопка «Скачать прайс (CSV)» формирует файл из актуальных товаров.

## Удобство админки
Поиск, фильтр по категории, фильтр наличия, сортировка (название/категория/цена/изменение), адаптив под телефон.

## Запуск локально
```bash
cd aquauzel-site && python3 -m http.server 8099
# открыть http://localhost:8099  и  http://localhost:8099/admin.html
```
(Админка требует заполненного `supabase-config.js`; без него `/admin.html` покажет инструкцию, а публичная часть — статический прайс.)

## Деплой
Коммит в `main` → GitHub Actions публикует на GitHub Pages (домен aquauzel.kz). Отдельной сборки не нужно — статика.

## Ограничения
- Нужен аккаунт Supabase (бесплатно) — иначе админку нельзя сделать безопасной на чистом GitHub Pages.
- PDF-прайс реализован через печать браузера (надёжно для кириллицы); прямая выгрузка — CSV. При желании можно добавить серверную генерацию PDF.
- Старый детальный матричный прайс упрощён до плоской модели (цена за позицию) — по согласованию.
