Настрой себя для этого проекта так, как описано ниже. Действуй автономно: не

## 1. Установи и подключи Graphify
- pip install graphifyy
- Проверь: graphify --help
- Подключи к Codex: graphify codex install
  (если не сработает — graphify install --platform codex)
- Если Graphify попросит перезапуск — перезапустись.

## 2. Построй граф знаний по проекту (без API‑ключа, только код)
- В корне проекта: graphify update .
  (AST‑only, ключ LLM НЕ нужен; создаст graphify-out/graph.json)
- Если update не сработает на «чистом» проекте — собери code‑only граф:
  1) создай временную папку;
  2) скопируй туда ТОЛЬКО код и важные конфиги (src, lib, app, server, web,
     android, ios, test, tools, scripts, *.js/.ts/.py/.go/.kt/.swift,
     package.json, pyproject.toml, pubspec.yaml и т.п.);
  3) НЕ копируй node_modules, .git, build, assets, изображения, видео, PDF,
     большие бинарники;
  4) graphify extract <временная_папка> --no-cluster
- Граф держи в graphify-out/. В .gitignore добавь строку:
  graphify-out/cache/
  (сам граф — graph.json/GRAPH_REPORT.md/graph.html — храни в репозитории).

## 3. Используй граф ПЕРЕД работой (это твоя knowledge‑graph база)
- Вопрос по проекту:   graphify query "вопрос" --graph graphify-out/graph.json
- Объяснить сущность:   graphify explain "ИмяКлассаИлиФункции"
- Связь между двумя:    graphify path "A" "B"
- Что затронут правки:  graphify affected "ИмяСущности"
Перед сложными изменениями сначала query/path/explain, потом редактируй.
Если граф даёт мало контекста — дополняй обычным чтением файлов.

## 4. Работай автономно (не спрашивай разрешение на каждый файл)
Настрой свой режим одобрения так, чтобы НЕ переспрашивать на каждое изменение,
а выполнять задания подряд. Для Codex CLI это «полный авто»:
- запуск: codex --full-auto
- либо в ~/.codex/config.toml:
      approval_policy = "never"
      sandbox_mode    = "workspace-write"
  (если для установки пакетов нужен доступ в сеть — sandbox_mode = "danger-full-access",
   понимая, что это снимает песочницу; включай осознанно).
Примени то, что подходит твоей текущей версии Codex. Цель — выполнять задачи
до конца без вопросов «можно ли изменить этот файл».

## 5. ВСЕГДА сохраняй после каждой доработки
После КАЖДОЙ доработки любого проекта:
  1) если менялся КОД — обнови граф: graphify update .
  2) git add -A && git commit -m "понятное сообщение"
  3) git push в рабочую ветку (и влей в main, если так принято в проекте)
Держи проект чистым: не коммить кэш и временные файлы; структурируй знания по
проектам — для каждого проекта свой graphify-out.

## 6. В конце подтверди настройку
